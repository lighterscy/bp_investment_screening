# 项目初步研判报告格式与目录

## 版式要求

- 纸张：A4。
- 页边距：上 3.7cm、下 3.5cm、左 2.8cm、右 2.6cm。
- 标题：居中，二号，方正小标宋简体。
- 正文：三号，仿宋_GB2312，首行缩进 2 字符。
- 一级标题：三号黑体。
- 二级标题：三号楷体_GB2312。
- 三级标题：三号仿宋_GB2312 加粗。
- 行距：固定值 28 磅。
- 数字与英文：Times New Roman。

## 技术背景子标题生成规则

“技术背景”部分的子标题不固定复用参考文档，而是根据 Layer1 中的“核心技术与技术壁垒”节点动态生成。系统优先调用配置的 LLM，未配置 LLM 时使用保守规则生成。

LLM 拆分标题时，只输入“核心技术与技术壁垒”节点，目标是为零基础读者搭建循序渐进的学习路径：先解释上位技术体系，再说明发展阶段和传统方案痛点，最后引出项目技术路线和验证重点。

未配置 LLM 时的 fallback：

```text
1、行业的产业链位置与核心应用场景
2、项目核心产品涉及的关键技术原理
3、现有技术路线的主要痛点与约束
4、项目方案的技术切入点与验证重点
```

LLM prompt 位于 `prompts/technical_background_outline.md`。
